## Como funciona

A ação **Enviar Webhook** em uma automação faz uma chamada HTTP (POST por padrão, GET também suportado) para uma URL externa quando o fluxo chega naquele nó. É diferente dos webhooks outbound (que disparam por eventos do CRM) — aqui você tem controle de quando o disparo acontece dentro do fluxo, mas **não** controle total do corpo da requisição: o corpo customizado é anexado ao payload padrão, não o substitui (veja [Payload padrão](#payload-padrão-sem-body-customizado) abaixo).

## Configurar a ação

<Steps>
  <Step title="Abra o editor de automações">
    Acesse **Automações** → crie ou edite uma automação existente.
  </Step>
  <Step title="Adicione um nó Ação">
    Clique em **+** no canvas e selecione **Ação** (nó laranja).
  </Step>
  <Step title="Selecione Enviar Webhook">
    No campo **Tipo de ação**, escolha **Enviar webhook**.
  </Step>
  <Step title="Configure a URL">
    Cole a URL de destino (HTTPS obrigatório).
  </Step>
  <Step title="Personalize o body (opcional)">
    Use variáveis para incluir dados do contexto. O JSON que você digitar aqui entra no payload padrão dentro do campo `data`:

    ```json
    {
      "contato": "{{name}}",
      "telefone": "{{phone}}",
      "etapa": "{{stage}}"
    }
    ```
  </Step>
</Steps>

## Variáveis disponíveis

As variáveis usam **chave plana** — notação com ponto (`{{contact.name}}`) não é interpretada e aparece literalmente na saída.

| Variável | Descrição |
|----------|-----------|
| `{{name}}` | Nome do contato |
| `{{phone}}` | Telefone do contato |
| `{{email}}` | E-mail do contato |
| `{{company}}` | Empresa do contato |
| `{{notes}}` | Notas do contato |
| `{{stage}}` | Nome da etapa atual do card no pipeline |
| `{{assigned_agent}}` | Nome do atendente responsável |
| `{{lastMessage}}` | Texto da última mensagem recebida |
| `{{webhook.<campo>}}` | Quando a automação foi iniciada por um gatilho **Webhook recebido**, qualquer campo escalar do payload de entrada (ex.: `{{webhook.product}}`, `{{webhook.amount}}`) |
| `{{<outputVar>}}` / `{{<outputVar>_<campo>}}` | Saída de um nó **HTTP** ou **Vistum IA** anterior no fluxo |

Referência completa em [Automações: Nós e Limites](/automations/nodes#variaveis).

## Payload padrão (sem body customizado)

```json
{
  "event": "automation.webhook",
  "automationId": "auto_xxx",
  "workspaceId": "wks_xxx",
  "timestamp": 1715093400,
  "contact": {
    "id": "cnt_xxx",
    "name": "Nome do Contato",
    "phone": "5511999887766"
  },
  "pipeline": {
    "cardId": "card_xxx",
    "fromStageId": "stage_xxx",
    "toStageId": "stage_yyy"
  },
  "trigger": "message_received",
  "vars": {}
}
```

<Note>
  `timestamp` é um **Unix timestamp inteiro** (segundos). Converta com `new Date(timestamp * 1000)` em JS ou `datetime.fromtimestamp(timestamp)` em Python.
</Note>

### Com body customizado

Se você preencher o campo **Personalizar body**, o JSON é interpolado e entra **dentro** do payload padrão, no campo `data` — os campos `event`, `automationId`, `contact`, `pipeline`, `trigger` e `vars` continuam sendo enviados normalmente:

```json
{
  "event": "automation.webhook",
  "automationId": "auto_xxx",
  "workspaceId": "wks_xxx",
  "timestamp": 1715093400,
  "contact": { "id": "cnt_xxx", "name": "Nome do Contato", "phone": "5511999887766" },
  "pipeline": { "cardId": "card_xxx", "fromStageId": null, "toStageId": null },
  "trigger": "stage_changed",
  "vars": {},
  "data": {
    "contato": "Nome do Contato",
    "telefone": "5511999887766",
    "etapa": "Proposta Enviada"
  }
}
```

<Warning>
  Se o body customizado não for um JSON válido depois da interpolação das variáveis, ele é descartado (fica de fora do payload) e o erro é registrado no log da execução — a chamada ainda é feita, só sem o campo `data`.
</Warning>

## Exemplos práticos

### Notificar n8n quando card mudar de etapa

1. Gatilho: **Mudança de etapa no pipeline** → etapa `Proposta Enviada`
2. Nó Ação: Enviar Webhook → URL do n8n
3. Body:
   ```json
   {
     "evento": "proposta_enviada",
     "cliente": "{{name}}",
     "telefone": "{{phone}}"
   }
   ```

### Criar registro no Google Sheets via n8n

1. Gatilho: **Novo contato criado**
2. Nó Ação: Enviar Webhook → URL do workflow n8n de planilha
3. O n8n recebe os dados e adiciona uma linha na planilha

### Alertar o time no Slack quando negócio for ganho

1. Gatilho: **Negócio marcado como ganho**
2. Nó Ação: Enviar Webhook → URL do n8n
3. O n8n posta mensagem no Slack: `🎉 {{name}} fechou negócio!`

## Limites e comportamento

- **Timeout**: 10 segundos por chamada, sem retry
- **Chamadas por execução**: máximo de **3 chamadas HTTP por execução do fluxo**, limite **compartilhado** entre a ação Enviar Webhook e o nó HTTP genérico — a partir da 4ª chamada (de qualquer um dos dois tipos), o nó é ignorado silenciosamente
- **Log**: cada execução aparece no histórico com URL mascarada, HTTP status e tempo de resposta
- **SSRF**: URLs de redes privadas são bloqueadas automaticamente (localhost, IPs internos), sem seguir redirects
